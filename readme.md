# TeamFinder - Платформа для поиска единомышленников

**TeamFinder** - платформа создана для разработчиков, дизайнеров, тестировщиков и других IT-специалистов. Она решает проблему поиска команды для совместной работы над Pet-проектами. Авторизованные пользователи могут публиковать свои идеи, заявлять о поиске определённых стеков технологий, добавлять чужие проекты в «Избранное», откликаться на предложения и находить контакты единомышленников.

---

## Основной функционал приложения

Веб-приложение полностью укомплектовано бэкенд-логикой на базе Django и интегрировано с готовым фронтендом. В проекте реализованы следующие возможности:
- **Аутентификация и профили:** Регистрация, вход, выход, смена пароля, а также просмотр и редактирование личного профиля пользователя (включая загрузку аватара).
- **Управление проектами:** Создание, просмотр подробной информации, редактирование и удаление карточек проектов авторами.
- **Пагинация и сортировка:** Списки актуальных проектов на главной странице отсортированы от новых к старым и разбиты на страницы по 12 карточек на каждой.
---

## Архитектура и Технический стек

При разработке бэкенда использовались современные практики и требования к промышленной разработке:
- **Язык программирования:** Python 3.12
- **Фреймворк:** Django 5.2.4
- **База данных:** PostgreSQL 15
- **WSGI-сервер:** Gunicorn
- **Контейнеризация:** Docker и Docker Compose
- **Качество кода:** Код полностью проверен линтерами на соответствие стандарту PEP 8.

---

## Инфраструктурные особенности

1. **Зависимости:** Все зависимости зафиксированы в файле `requirements.txt`.
2. **Сохранение данных (Persistent Volumes):** Настроены Docker-хранилища:
   - `postgres_data` - для сохранения учетных записей и проектов даже после полного удаления контейнеров.
   - `static_volume` - для собранной статики фронтенда.
   - `media_volume` - для загружаемых пользователями аватаров и изображений.
4. Для раздачи статических файлов используется веб-сервер Nginx, который считывает собранную статику через привязанную директорию `./static_backend`.

## Подробная инструкция по запуску и развертыванию

Для локальной проверки или деплоя проекта на сервер выполните следующие шаги:

### Шаг 1. Проверка системных требований
Убедитесь, что в вашей системе установлены компоненты среды контейнеризации. Проверить это можно в терминале:
  - `docker --version`
  - `docker-compose --version`

### Шаг 2. Клонирование репозитория и проверка структуры

На данном этапе необходимо перенести исходный код проекта на целевой сервери убедиться в наличии всех конфигурационных файлов, необходимых для сборки Docker-окружения.

**Склонируйте ваш репозиторий** с проектом TeamFinder и перейдите в его корневую директорию:
   - `git clone https://github.com/Hokadva/team-finder-ad.git`
   - `cd team-finder`

### Шаг 3. Создание конфигурационного файла переменных окружения\

1) Поскольку Docker-контейнеры изолированы, для безопасной передачи параметров авторизации и связывания Django с СУБД PostgreSQL необходимо создать локальный файл окружения.
  - `nano .env`

2) Вставьте в файл конфигурационные параметры (пример наполнения):
```
  DJANGO_SECRET_KEY=django-secret-key
  DJANGO_DEBUG=False
  ALLOWED_HOSTS=localhost,127.0.0.1,0.0.0.0
  DJANGO_ALLOWED_HOSTS=*
  POSTGRES_DB=teamfinder
  POSTGRES_USER=admin-admin
  POSTGRES_PASSWORD=secure-password
  POSTGRES_HOST=db
  POSTGRES_PORT=5432
```
3) Сохраните изменения

### Шаг 4. Сборка Docker-образов и запуск проекта

Выполните сборку и запуск контейнеров:
  - `docker-compose up --build -d`

### Шаг 5. Контроль запуска и верификация логов

Проверьте логи сервиса web:
  -`docker-compose logs web`
